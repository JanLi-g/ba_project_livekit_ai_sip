## 📋 EMPFEHLUNGEN FÜR DAS GLOSSAR

### Priorisierung (Wichtigkeit für die BA):

#### **Höchste Priorität (Kritisch):**
1. **Latenz** - wird in allen Kapiteln verwendet, besonders wichtig für Evaluation
2. **Transkodierung** - zentral für Media-Verarbeitung
3. **Codec** - häufig erwähnt bei Audio-Verarbeitung
4. **Routing** - fundamental für Netzwerk-/Telefonkommunikation
5. **Signalisierung** - Basis des SIP-Protokolls

#### **Hohe Priorität (Wichtig):**
6. **Jitter** - verwendet bei Audioqualität
7. **Trunk** - essentiell für Provider-Anbindung
8. **Bridge** - zentrale Komponente des Systems
9. **Token** - verwendet bei Authentifizierung
10. **Monitoring** - relevant für Betrieb

#### **Mittlere Priorität (Nützlich):**
11. **Subprozessor** - wichtig für Datenschutz-Kontext
12. **Endpoint** - häufig erwähnt
13. **Gateway** - erweitern vorhandenen Eintrag
14. **Dialplan** - technischer Begriff
15. **Fallback** - verwendet in mehreren Kontexten

## 📝 DETAILLIERTE GLOSSAR-EINTRÄGE (ZUM HINZUFÜGEN)

```latex
% Neue Fachwörter für glossary.tex

\newglossaryentry{gls-Latenz}{
  type=fachwort,
  name=Latenz,
  description={Zeitverzögerung zwischen einem Eingabeereignis und der entsprechenden Systemreaktion oder Ausgabe, gemessen in Millisekunden (ms). In Telekommunikation kritisch für Nutzerwahrnehmung und Gesprächsqualität. Typische Schwellwerte: <150 ms unmerklich, <400 ms akzeptabel, >400 ms störend (ITU-T G.114)}
}
\glsadd{gls-Latenz}

\newglossaryentry{gls-Codec}{
  type=fachwort,
  name=Codec,
  description={Encoder-Decoder-Verfahren zur Digitalisierung, Kompression und Dekomprimierung von Audio- oder Videodaten. Beispiele: G.711 (klassische Telefonie), Opus (WebRTC), PCM16 (Spracherkennung)}
}
\glsadd{gls-Codec}

\newglossaryentry{gls-Transkodierung}{
  type=fachwort,
  name=Transkodierung,
  description={Konvertierung von codierten Daten (insbesondere Audio/Video) von einem Format oder Codec in einen anderen, z.B. G.711 zu Opus. Notwendig, wenn unterschiedliche Systeme kompatible Audio-Formate benötigen}
}
\glsadd{gls-Transkodierung}

\newglossaryentry{gls-Routing}{
  type=fachwort,
  name=Routing,
  description={Verfahren zur intelligenten Weiterleitung von Datenpaketen oder Telefon-Anrufen von einer Quelle zum Ziel über ein Netzwerk, basierend auf definierten Regeln oder Routingtabellen}
}
\glsadd{gls-Routing}

\newglossaryentry{gls-Signalisierung}{
  type=fachwort,
  name=Signalisierung,
  description={Steuersignale und Nachrichten zur Verwaltung von Verbindungen, einschließlich Aufbau, Verwaltung und Abbau. In VoIP realisiert durch SIP-Protokoll}
}
\glsadd{gls-Signalisierung}

\newglossaryentry{gls-Jitter}{
  type=fachwort,
  name=Jitter,
  description={Schwankung oder Varianz in der Ankunftsverzögerung von Datenpaketen in einem Netzwerk. Hoher Jitter kann zu Audio-Verzerrungen oder Qualitätseinbußen führen}
}
\glsadd{gls-Jitter}

\newglossaryentry{gls-Trunk}{
  type=fachwort,
  name=Trunk,
  description={Verbindung zwischen einer lokalen Telefonanlage (PBX) und einem externen Telefonanbieter (Provider) oder einer anderen Telefonanlage. Trägt mehrere Anrufe simultan}
}
\glsadd{gls-Trunk}

\newglossaryentry{gls-Bridge}{
  type=fachwort,
  name=Bridge,
  description={Netzwerk- oder Softwarekomponente, die zwei unterschiedliche Netzwerke, Protokolle oder Systeme verbindet und Datenaustausch ermöglicht, z.B. SIP-WebRTC-Bridge}
}
\glsadd{gls-Bridge}

\newglossaryentry{gls-Token}{
  type=fachwort,
  name=Token,
  description={Eindeutiger, temporärer Authentifizierungsschlüssel mit limitierter Gültigkeitsdauer, verwendet zur Authentifizierung von Benutzern oder Services ohne Übergabe von Passwörtern}
}
\glsadd{gls-Token}

\newglossaryentry{gls-Monitoring}{
  type=fachwort,
  name=Monitoring,
  description={Kontinuierliche Überwachung von Systemmetriken, Ereignissen und Logs zur Sicherung der Verfügbarkeit, Performance und Fehleridentifikation}
}
\glsadd{gls-Monitoring}

\newglossaryentry{gls-Subprozessor}{
  type=fachwort,
  name=Subprozessor,
  description={Externe Dienstleister, der im Auftrag eines Verantwortlichen oder Auftragsverarbeiters personenbezogene Daten verarbeitet (DSGVO Art. 28). Muss transparent offengelegt und vertraglich dokumentiert sein}
}
\glsadd{gls-Subprozessor}

\newglossaryentry{gls-Endpoint}{
  type=fachwort,
  name=Endpoint,
  description={Endpunkt einer Netzwerkverbindung, z.B. ein SIP-Telefon, WebRTC-Client oder Server, der Daten sendet oder empfängt}
}
\glsadd{gls-Endpoint}

\newglossaryentry{gls-Dialplan}{
  type=fachwort,
  name=Dialplan,
  description={Regelsatz in einer Telefonanlage (PBX), der definiert, wie Anrufe basierend auf gewählten Nummern oder anderen Kriterien geroutet werden}
}
\glsadd{gls-Dialplan}

\newglossaryentry{gls-Fallback}{
  type=fachwort,
  name=Fallback,
  description={Ausweichmechanismus oder alternative Strategie, die aktiviert wird, wenn ein primärer Mechanismus oder Prozess fehlschlägt}
}
\glsadd{gls-Fallback}

\newglossaryentry{gls-Whitelist}{
  type=fachwort,
  name=Whitelist,
  description={Liste von erlaubten IP-Adressen, Benutzern oder anderen Entities. Nur Einträge auf der Whitelist werden akzeptiert, alle anderen werden blockiert}
}
\glsadd{gls-Whitelist}
```
